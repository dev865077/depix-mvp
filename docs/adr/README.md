# ADRs

Esta pasta existe para centralizar Architecture Decision Records do projeto.

## Formato padrao

Cada ADR deve conter:

- Contexto
- Decisao
- Alternativas consideradas
- Consequencias
- Status

## Estado atual

O projeto ja tem varias decisoes travadas em documentos gerais, mas ainda nao consolidou esses pontos em ADRs formais. Esta pasta prepara essa camada sem misturar ADR com nota solta.
