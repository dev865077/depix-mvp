# ADRs

Esta pagina centraliza a politica de `Architecture Decision Records` do projeto.

## Formato padrao

Cada ADR deve conter:

- Contexto
- Decisao
- Alternativas consideradas
- Consequencias
- Status

## Estado atual

O projeto ja tem varias decisoes travadas em documentos gerais, mas ainda nao consolidou esses pontos em ADRs formais independentes.

## Proximo passo natural

Os primeiros ADRs candidatos sao:

- Worker unico no MVP
- `tenantId` como fronteira logica obrigatoria
- `D1` como banco unico do MVP
- webhook da Eulen como confirmacao primaria
